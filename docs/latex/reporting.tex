\section{Report Structure}
Every generated report contains the following sections in order:
\begin{enumerate}
  \item Title page (project name, date, classification)
  \item Project metadata (ID, authorization scope, analyst)
  \item Authorization statement
  \item Executive summary
  \item Scope of assessment
  \item Inventory summary (target devices)
  \item Findings table with severity matrix
  \item Evidence references (with SHA-256)
  \item Analyst notes
  \item Recommendations
  \item Appendix
\end{enumerate}

\section{Export Workflow}
\begin{enumerate}
  \item Navigate to report detail page
  \item Click \textbf{Print / Export to PDF} button
  \item Browser print dialog opens with print CSS applied
  \item Select ``Save as PDF'' destination
  \item Report exports as single-file PDF with all styling
\end{enumerate}

\section{API Endpoints}
\begin{lstlisting}
POST /reports/generate   - Generate report for project
GET  /reports            - List all reports
GET  /reports/{id}       - Get report metadata
GET  /reports/{id}/view  - View rendered HTML report
\end{lstlisting}
